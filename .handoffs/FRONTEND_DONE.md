## Post-QA Fixes

### Fix: `useSearchParams()` Suspense boundary on `/login` (QA defect)

**Defect:** Next.js production build failed with:
> `useSearchParams() should be wrapped in a suspense boundary at page "/login"`

**Root cause:** `src/app/login/page.tsx` was a single `'use client'` component that called `useSearchParams()` directly at the page level. Next.js requires any component calling `useSearchParams()` to be wrapped in a `<Suspense>` boundary to avoid blocking static rendering.

**Fix applied:**

1. **Created** `src/components/auth/LoginForm.tsx` — extracted the full login component (OTP state machine, Google OAuth, redirect logic, all JSX) into a dedicated client component. This is where `useSearchParams()` now lives.

2. **Rewrote** `src/app/login/page.tsx` — converted to a server-compatible page shell (no `'use client'`, no hooks). It imports `Suspense` from React and wraps `<LoginForm />` in a `<Suspense fallback={<LoginFallback />}>`. The fallback renders a skeleton layout matching the card dimensions.

**Files changed:**
- `src/app/login/page.tsx` — now a thin server shell with Suspense wrapper
- `src/components/auth/LoginForm.tsx` — new file containing all login logic
