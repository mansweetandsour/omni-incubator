## How to Run Locally

1. Copy `.env.local.example` to `.env.local` and fill in credentials
2. `npm run dev` — starts on http://localhost:3000
3. `npm run build` — production build (requires valid env vars for Sentry sourcemap upload)

Apply database migrations via Supabase CLI:
```bash
supabase db push
# or for a fresh project:
supabase db reset
```

## Spec Deviations

1. **`next.config.ts` Sentry options**: SPEC.md listed `hideSourceMaps: true` and `widenClientFileUpload: true` which are not valid in `@sentry/nextjs` v9 (the version installed). These were replaced with `sourcemaps: { disable: ... }` which is the correct v9 API. The functional intent (suppress sourcemap upload when no auth token is configured) is preserved.

2. **Placeholder pages use raw Tailwind classes**: shadcn/ui was not initialized in this pass (it requires interactive CLI prompts). Placeholder pages use minimal Tailwind styling. shadcn/ui can be initialized in the Frontend agent pass with `npx shadcn@latest init` — it only touches `src/components/ui/` and `globals.css`.

3. **Login page and profile form use minimal Tailwind styling**: Per the above. Full shadcn/ui component styling (Button, Input, Card, etc.) will be applied in the Frontend agent pass.

4. **NavbarAuth sign-out/dropdown is simplified**: Full shadcn/ui DropdownMenu with "Profile, My E-books, Orders, Entries, Subscription, Sign Out" is Frontend territory. The server-side auth gate and sign-out RPC are fully implemented.
